# 🇨🇳 简体中文

> 📢 **当前状态：Beta 公测中**
>
> 核心代码目前为 **非公开** 状态。本仓库主要用于问题反馈（Issues）、功能建议（Feature Requests）以及高级协作申请。

## ✨ 项目特性

* **统一商业系统**：官方商城（B2C）与玩家市场（C2C）共享同一套账号、钱包与订单底层架构。
* **Web 优先架构**：内置响应式玩家端与管理后台。支持便捷的内置 HTTP 模式，亦可快速对接外部 Nginx/CDN 托管。
* **双货币体系**：原生支持 `ShopCoin` 与 `GameCoin` 双货币，并支持通过 Vault 对接现有经济插件。
* **完整交易闭环**：包含自动发货、手动领取（`/ws claim`）、退款、兑换码与团购券营销系统。
* **后台与审计工具**：提供细粒度权限、账户管理功能与全局操作审计日志支持。

## 🛠 反馈指南

欢迎所有公测用户参与完善 WebShopX。为了让 Bug 得到快速定位，请配合：
1. **先搜索**：提交前请先检索已有 Issue，避免重复提交。
2. **用模板**：请务必选择对应的 Issue 模板。
3. **给环境**：请提供完整的运行环境信息：
   * Java 版本
   * 服务端类型与版本
   * 完整的控制台堆栈报错日志（Stacktrace）

# 🇺🇸 English

> 📢 **Current Status: Public Beta**
> 
> The core source code is currently **closed-source**. This repository is strictly dedicated to issue tracking, feature requests, and developer/pro-user applications.

## ✨ Features

* **Unified Commerce System**: Official stores (B2C) and player markets (C2C) share the same underlying account, wallet, and order infrastructure.
* **Web-First Architecture**: Built-in responsive player frontend and admin dashboard. Supports embedded HTTP mode or external Nginx/CDN deployment.
* **Dual Currency Support**: Native support for `ShopCoin` and `GameCoin`, with seamless Vault economy bridge.
* **Complete Transaction Workflow**: Covers automatic delivery, manual claiming (`/ws claim`), refunds, redeem codes, and coupon/group-buy systems.
* **Administrative & Audit Tools**: Fine-grained permissions, comprehensive account management tools, and detailed audit logging.

## 🛠 Feedback Guidelines

We welcome all beta testers to help improve WebShopX:
1. **Search First**: Check existing Issues to avoid duplicates.
2. **Use Templates**: Use the correct Issue template for your request.
3. **Environment Info**: Provide complete environment details:
   * Java version (**Supports Java 17 - 25**, Java 21 / 25 recommended)
   * Server software/version
   * Full console logs/stack traces

## 👨‍💻 Developer & Pro-User Collaboration

WebShopX may transition toward a more open development model in the future. For now, we are inviting a limited number of advanced testers and contributors.

> ⚠️ **Notice**: Regular users seeking standard support should use standard Issues or community groups instead of applying.

### Suitable Applicants
* **Pro Users**: Capable of repeated testing, reproducing complex edge-case bugs, and helping improve docs/workflows.
* **Developers**: Familiar with Java, Spigot/Paper API, and modern collaborative Git workflows.

### How to Apply
Create a new Issue and select the **"Developer/Pro-User Application"** template.
